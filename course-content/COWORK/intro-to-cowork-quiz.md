## Question 1

**What do subagents enable when Cowork is handling a large task with independent pieces?**

- Run tasks on other computers in your network
- Automatically retry failed steps without telling you
- **✓ Work on separate pieces at the same time, each with its own focused context**
- Store each piece of work permanently so it can be reused later

## Question 2

**How does a Cowork project's memory differ from memory in Claude chat?**

- Cowork project memory is larger and can hold more information
- **✓ A Cowork project's memory stays inside that project, while chat memory applies across all your conversations**
- They work the same way and share the same stored information
- Cowork project memory is read-only and cannot be updated during a task

## Question 3

**Which statement correctly describes the relationship between skills and plugins?**

- Skills are for developers and plugins are for end users

## Question 4

**A scheduled task was due while your computer was asleep. What happens?**

- The task is skipped and will run at the next scheduled time
- The task fails and you need to reschedule it manually
- The task runs in the cloud and the result is waiting when you wake the computer
- **✓ Cowork runs the task as soon as you're back and lets you know it was delayed**

## Question 5

**Each Cowork task starts fresh. How does context carry from one task to the next?**

- **✓ Through a project (folder, instructions, and memory for one piece of work) and global instructions that apply to every task**
- Cowork automatically remembers every conversation and applies it to new tasks
- You export the previous task's output and paste it into the new prompt
- Context only carries if you keep the same chat window open

## Question 6

**What's the recommended habit before sending a Cowork output onward or acting on it?**

- Run the same task a second time and compare the two outputs
- Ask Claude to grade its own work on a scale of one to ten
- **✓ Review it yourself, since a polished result can still be wrong**
- Trust the result if Claude didn't flag any issues during the task

## Question 7

- The folder contents are copied to Anthropic servers for processing
- All files in the folder are automatically backed up before each task
- **✓ Claude can read, create, and edit files there directly, without an upload or download step**
- Claude gets read-only access and saves any outputs to a separate location

## Cornell adaptation notes

### Mapping the must-stick takeaways to Module 3 slides

These 8 questions are Anthropic's own filter on what every Cowork learner should walk away with. Most reinforce anchors already in our Module 3 plan; two add usefully.

**Already in the Module 3 anchor list — quiz confirms importance:**

- **Q7 ("Point Claude at a folder") → Module 3 slide 1.** *The* paradigm shift. Quiz puts it front-and-center too.
- **Q8 (Cowork vs. Chat task fit) → Module 3 slide on "three task-fit patterns."** The quiz's example — *"turning a folder of meeting notes and spreadsheets into a finished presentation"* — is a clean test case for what Cowork does that Chat doesn't.
- **Q3 (Skills vs. plugins) → Module 3 slide on the four building blocks.** Especially the precise distinction: skills are cross-surface; plugins are the Cowork-specific bundling of skills + connectors + subagents.
- **Q5 (Context carries via project + global) → Module 3 slide on Global Instructions + Projects.** The quiz answer is the slide's anchor line: each task starts fresh; project + global is how persistence works.
- **Q6 (Review before acting) → Module 3 closing slide.** Same Diligence message as AIF's "stand behind every word" — and the language is even simpler: *"a polished result can still be wrong."* Worth lifting verbatim.

**New additions worth folding into Module 3:**

- **Q2 (Project memory vs chat memory).** This is a subtle but operationally important distinction we hadn't flagged separately. **Cornell-specific implication:** memory in a sponsored-programs Project doesn't bleed into other work. That's a privacy story worth a sentence in the safety section — *project memory stays scoped to its project.*
- **Q4 (Scheduled task while computer asleep).** Practical detail: Cowork resumes delayed tasks when the computer is back, doesn't skip or fail silently. Worth a one-line callout on the `/schedule` slide so learners don't get burned by "I missed my run" anxiety.

### Verbatim wording worth keeping

A few quiz answers are unusually clean. Candidates for direct lift into Module 3 slide copy:

- *"A polished result can still be wrong."* — pairs with Cornell's "stand behind every word" mantra from Module 2.
