Quick refresher

Two things to know about AI itself.

Generative vs traditional

Claude creates something new

Traditional AI analyzes and classifies existing data. Generative AI (Claude) creates new text, drafts, summaries. That’s why output isn’t deterministic — it’s probabilistic.

Working memory

The context window

Claude holds a finite amount of conversation + uploaded content at once. Long proposal? Multiple RFPs? Scope the conversation. Projects help carry context across chats.

The framework

The 4 D’s.

Delegation

Decide what to hand over

Setting goals and deciding whether, when, and how to engage with AI.

Description

Communicate clearly

Effectively describing goals to prompt useful AI behaviors and outputs.

Discernment

Evaluate critically

Accurately assessing the usefulness of AI outputs and behaviors.

Diligence

Take responsibility

For what we do with AI and how we do it.

How they connect

Two loops.

Loop 1

Delegation ↔ Diligence

When and whether to use AI. Should I use AI here? Did I take responsibility for the result?

Loop 2

Description ↔ Discernment

How to use AI day-to-day. Did I describe it well? Did I read what came back critically?

Today: three filters help you answer Loop 1. Then we go deep on Loop 2.

Filter 2 · task type

“Should AI do this?” — not “Can AI do this?”

AI handles

Standardized work

Documented info, clear processes. Look up a CFR section. Reformat a budget table.

AI assists

Draft + you review

AI drafts, you review before sending. Justification narratives, sponsor briefs, internal memos.

Human handles

Judgment, relationships, precedent

Sponsor calls, audit findings, allowability decisions, anything precedent-setting.

Loop 2 · how

Description and Discernment.

Cornell’s 3-part prompt from Module 1 maps cleanly to AI Fluency’s Description triad.

Description (how you ask)

3-part prompt

  • Context = Performance Description (tone, role)
  • Task = Process Description (approach)
  • Format = Product Description (output)
Discernment (how you read it)

Three critical checks

  • Is the output good? (Product)
  • Did it use the right sources? (Process)
  • Did it push back when uncertain? (Performance)

All four D’s in motion

Exercise A revisited.

Last week’s eCFR lookup was already a 4D exercise. Let’s name the parts:

  • Delegation — Choose Claude 3P (Yellow data, Platform Awareness).
  • Description — “Look up 2 CFR 200, Subpart E. Cite section numbers. Plain-English summary for a non-expert PI.”
  • Discernment — Did the section numbers match? Did the summary cover all subparts? Anything missing?
  • Diligence — Read the original sections before sending to a PI. Disclose AI involvement if it matters.

Hands-on · 25 min

Test against data you already know cold.

You brought a dataset you understand — an annual award report, a portfolio review, a closed proposal cycle, a compliance dashboard.

  1. Open Claude 3P with the dataset in scope.
  2. Ask a question you already know the answer to.
  3. Compare. If it matches, ask how it got there. If it doesn’t, refine the prompt — or find the limit.

The takeaway: before you trust AI with new data, validate on data you already know. That’s Deployment Diligence in practice.

From AIF-NP Lesson 6, “Data analysis with AI.”

Diligence · the values anchor

Stand behind every word.

When AI-assisted work goes to a PI, an OSP colleague, or a sponsor, you own every claim. Not “the AI said so.” You.

That’s not a new rule for sponsored programs work — it’s the rule that’s always applied. The framing helps name it.

And the “human in the loop” piece: being the human in the loop means ensuring AI serves your research mission and the PI / sponsor / colleague relationships that define your work.
