Data rules

Same rules as Sandbox AI.

  • OK to use: confidential Cornell data used in research administration — protocols, awards, proposals, IP.
  • Not allowed: HIPAA data (we shouldn’t encounter it) and export-controlled or government-restricted information.
  • Claude 3P is a different front end pointing at the same infrastructure as Sandbox AI — the LiteLLM gateway to the cloud models Cornell already has data-protection contracts with.
  • Web search, when used, runs through Gemini’s enterprise grounding — your prompt is not logged.

If you can put it in Sandbox AI, you can put it in Claude 3P. Same gateway, same contracted infrastructure.

How AI works

Probabilistic, not deterministic.

An LLM builds its answer one token at a time, picking among plausible next words by probability. There is no fixed right answer — only what’s most likely given what came before.

That’s why two runs of the same prompt can come back worded differently, and occasionally with different substance. It’s a feature, not a bug — but it’s why you read every answer critically and spot-check the things that matter.

The 3-part prompt

Set Stage. Define Task. Specify Rules.

A simple frame that lifts almost any prompt. Every great prompt has these three parts — sometimes implicitly, sometimes in any order.

1 · Stage

Set Stage

Who you are, what you’re working on, who the output is for. The context Claude needs.

2 · Task

Define Task

What you want Claude to do, broken into specifics. Numbered lists for compound work.

3 · Rules

Specify Rules

Format, length, sources, tone, constraints. The “how,” not the “what.”

Adapted from Anthropic’s AI Fluency materials.

The 3-part prompt · Weak

This won’t get you what you want.

Write a market analysis report about streaming.
  • No stage — whose market? For what audience? What kind of streaming?
  • Vague task — what should the report cover? What sections?
  • No rules — how long? What format? What sources?

Claude will produce something, but it’ll be generic and you’ll spend the next hour re-prompting.

The 3-part prompt · Strong

All three parts. Same task, different result.

Create a market analysis report on the US indie film streaming opportunity. Include market size, growth trends, key competitors, consumer preferences, revenue models, and market gaps. Use current web research with citations. Structure as a professional report for an investor deck for a new indie streaming app.
1. Set Stage 2. Define Task 3. Specify Rules

Stage, Task, and Rules don’t have to appear in order — they just have to all be present.

The interface

What you see when you open Claude 3P.

  • Chat box. Where you type your prompt and drop in files.
  • Model picker (bottom). Sonnet for most work, Opus for hard reasoning, Haiku for quick lookups.
  • File drop. Drag and drop — PDFs, images, Word, Excel, CSV, code files all work.
  • Sidebar. Your previous chats, stored on your local machine until you delete them.
