The paradigm shift

Point Claude at a folder.

The one-sentence answer to “what’s different about Cowork?”

Claude can read, create, and edit files there directly — without an upload or download step.

Everything else in Module 3 — Skills, Plugins, subagents, scheduled tasks — is mechanics on top of that core affordance.

How Cowork moves

Plan · Execute · Connect.

The shape of the exchange: you describe an outcome. Claude plans the steps, works through them, and produces the deliverable — a file on your drive, not text to paste somewhere else.

Plan

Claude shows its approach

For multi-step work, you see the plan before execution. Review, adjust, approve.

Execute

Work runs on your machine

In an isolated environment. Long tasks can run while you do something else.

Connect

Reaches your systems

Email, drives, Slack, Teams, the tools you’re already signed into.

When to reach for Cowork

Three task-fit patterns.

Cowork shines on:

  • Multi-step tasks — gather → compare → draft → format. Hand it the whole arc, watch the Progress panel.
  • File-based tasks — inputs and outputs are real files on disk. Cowork reads, edits, saves directly.
  • Multi-tool tasks — spans email + drive + spreadsheet + a CRM. Cowork plans across them, executes as one delegation.

For a quick brainstorm or a single rewrite, reach for plain Claude Chat. For “turn a folder of notes into a finished presentation saved to my drive” — that’s Cowork.

Projects in practice

Memory that stays inside the project.

A Cornell example: a Sponsor Renewal Pipeline project pointed at the relevant award folder.

  • Project-specific instructions: how this sponsor likes briefs formatted, deadline patterns, key contacts.
  • Files: prior submissions, sponsor announcements, internal review notes.
  • Memory: past decisions (“we don’t budget for X with this sponsor”), persistent across many chats.

Privacy note: a Project’s memory stays inside that Project. Sponsor-A workstream context doesn’t bleed into sponsor-B work.

Skills · the deep dive

Your recurring workflows, taught to Cowork.

A Skill is a folder of assets that teaches Cowork how to run a specific process. Four possible parts:

  • SKILL.md — the runbook: trigger phrases + step-by-step logic. Required.
  • Assets — logos, master templates, branding.
  • References — “what good looks like” exemplars.
  • Scripts — small code files for repeated technical operations.

Cornell example: a cfr-lookup-for-pi skill — trigger phrase, instructions to use the AI4RA eCFR MCP server, the plain-English summary template, sample reference doc. Auto-activates.

Building one: prompt “I want to build a skill for [process]. Walk me through what you need to know.” Claude interviews you and generates the folder.

Plugins · install today, build together over time

Bundled expertise.

A Plugin packages multiple skills + connectors + subagents into one role-based install.

Pipeline

End-to-end

Sequences skills into a single complex delegation. E.g., a Financial Close plugin: pull data → process variances → draft executive memo.

Toolkit

Slash commands

Independent skills invoked by slash command. E.g., a Legal plugin with /nda-review and /clause-library.

Install today: Customize → Plugins, or /setup-cowork for a guided recommendation.

For Cornell research admin, over time: RAIS and the AI Innovation Hub partner with the cohort to build Cornell-specific plugins — bundling contributed skills, the AI4RA prompt library, and Cornell connectors. Contribute skills; RAIS publishes the plugin.

Trust your skill

Evals before you share.

Before publishing a skill for the cohort, validate it.

The skill-creator runs a built-in side-by-side comparison: with skill vs without skill. Verify the skill adds value over the baseline.

Tune one constraint at a time. Most skills stabilize in 1–2 rounds.

Working safely

Cornell overlay.

Cowork’s generic guidance: isolate folders, preserve masters, draft-and-test, avoid destructive verbs.

Cornell overlay — recall the Green / Yellow / Red model:

Green

Use freely

Public data: RFPs, CFR, published policies.

Yellow

Use with your guide

Confidential admin data: drafts, internal memos. Run through Cowork with your guide as Global Instructions.

Red

STOP

FERPA / HIPAA / export-controlled / sponsor-restricted — requires explicit Cornell sign-off, no matter how convenient Cowork looks.

Project memory stays inside its project. Scheduled tasks resume when the computer wakes — no anxiety about missed runs.

Hands-on · 25 min

Configure → Scaffold → Run.

Three-part live build. You leave with a working Project, a working Skill, and one task delegated.

  • (a) Configure · ~7 min — paste your guide as Global Instructions. Start a Project pointed at your folder.
  • (b) Scaffold · ~10 min — build a first Skill from one of the “AI can handle” tasks via the skill-creator interview. Rough draft, not polished.
  • (c) Run · ~8 min — issue the task with “Ask before acting” on. Watch the Progress panel. Review the output as a draft.

A polished, eval-tested skill takes more than 10 minutes. What you’re doing here is scaffolding the structure and the trigger. Refinement happens later, with RAIS / AI Innovation Hub support.
