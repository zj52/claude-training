## Key takeaways worth porting into our materials

1. **The Claude product lineup, with 3P highlighted.** This is the Cornell-specific framing the user flagged. We already have `slides/assets/ai-landscape.svg` doing similar work for the broader Cornell AI ecosystem — but a Claude-product-specific landscape (Projects · Artifacts · Skills · Connectors · Cowork) showing where 3P sits would be additive. **TODO:** consider building a "Claude products" diagram for Foundations slide 3 or 4 with 3P highlighted.

2. **The "Input → Transformation → Output" prompt pattern.** This is the user's existing teaching shorthand. It's a sibling to the AIF Description triad (Product / Process / Performance) and to our "context · task · format" 3-part prompt. Three frames for the same idea. Module 2 slide 7 should acknowledge all three terminologies and pick one for consistency.

3. **Skills vs. Projects in one line:** *projects store knowledge, skills perform tasks.* Quotable. Belongs in Module 3 (Cowork) as the anchor distinction.

4. **Enterprise Search as "a pre-built Project for your entire organization."** Useful mental model for the cohort if/when "Ask Cornell" rolls out. Worth a callout in Module 3 if relevant.

5. **The "do you have access to Gmail?" check.** A practical Diligence habit — *check what the tool can reach before you assume it can.* Belongs in Module 1 (Foundations) as a usability tip.
